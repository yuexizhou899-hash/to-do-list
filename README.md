# Purpose of the software
This to‑do list web application adopts the Agile software development process, which is selected over the Waterfall model due to its high flexibility, short iterative cycles, and suitability for small‑scale projects; unlike the rigid, linear Waterfall approach that restricts late adjustments.

Agile allows continuous testing, timely modifications, and rapid delivery of core features to match our simple task management needs. The software is designed as a lightweight, browser‑based daily task organizer.

The target market covers students, office workers, and general individual users who require convenient tools to record daily tasks, track to‑do items, mark completed work, and manage personal schedules without complex installation.

# Algorithm
1. The frontend fetches all tasks from the backend API when the page loads.
2. Displays tasks in a list format.
3. Sends a POST request to add a new task.
4. Sends a DELETE request to remove a task.
5. The UI refreshes automatically after each operation.
